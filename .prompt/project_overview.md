# 项目概览

## 项目结构

本文件描述项目集群的整体规划。每个子项目独立维护一份，复制到对应子目录后按需修改删减。

当前规划 4 个子项目：

### 1. 校园生活 PC 端

- 技术栈：Vue3 + Vue 生态相关库
- 说明：桌面端管理界面

### 2. 校园生活 H5 端

- 技术栈：Vue3 + Vue 生态相关库
- 说明：移动端 H5 页面

### 3. 学生门户管理端（PC 端，H5 页面响应式）

- 技术栈：Vue3 + Vue 生态相关库
- 说明：PC 端管理界面，同时做 H5 页面的响应式适配

### 4. 学生服务（服务端口）

- 技术栈：Node.js + Express + TypeScript + Prisma + PostgreSQL
- 说明：后端 API 服务

## API 结构（student-api 定调）

### 基础 URL

```
http://localhost:3000/api/v1
```

### 模块划分

| 类别 | 模块 | 端点前缀 | 说明 |
|------|------|----------|------|
| 核心 | 认证 | `/auth` | 登录、注册、Token 刷新 |
| 核心 | 学生 | `/students` | 学生信息 CRUD、批量导入 |
| 核心 | 班级 | `/classes` | 班级 CRUD、学生分配 |
| 核心 | 课程 | `/courses` | 课程管理、排课 |
| 核心 | 成绩 | `/grades` | 成绩录入、统计报表 |
| 核心 | 用户管理 | `/users` | 管理员/教师账号管理 |
| 拓展 | 模块注册 | `/extensions` | 模块注册、状态管理 |
| 拓展 | 业务模块 | `/extensions/:module` | 各拓展模块的统一入口 |

> 拓展模块采用可插拔设计，新增模块时只需在数据库 `modules` 表注册，API 层按命名规范自动挂载路由。前端对应在"校园生活"导航下动态渲染菜单。

### 接口风格

- RESTful 风格
- 统一响应格式：
  ```json
  {
    "code": 200,
    "message": "success",
    "data": {}
  }
  ```
- JWT Token 认证，Bearer 方式传递

### 拓展说明

- **modules 表**：统一管理所有拓展模块，前端动态读取生成导航菜单
- **module_data 表**：通用 JSONB 存储，适合快速原型，数据量大了再拆独立表
- **独立业务表**：如 `canteens` / `daily_menus` / `daily_books` / `daily_games` 等，按需创建
- 所有拓展表通过 `student_id` 与学生信息关联，共享用户体系

## 前端模块化设计思路

```
页面导航结构（预留校园生活入口）
├── 核心功能
│   ├── 学生管理
│   ├── 班级管理
│   ├── 课程管理
│   ├── 成绩管理
│   └── 系统设置
├── 校园生活  ← 导航预留，动态渲染
│   ├── 食堂菜单  （来自 modules 注册）
│   ├── 每日书籍  （来自 modules 注册）
│   ├── 每日游戏  （来自 modules 注册）
│   └── ...
└── 更多功能  ← 预留扩展位
```

- 前端侧边栏/导航菜单数据从 API 的 `/extensions` 接口动态获取
- 新增模块时只需后端注册 + 前端新增对应页面组件
- 三个前端项目共用同一套 API，各自渲染适合自己端口的 UI

```
projectXX/
├── （项目代码目录）

│   └── skills/
```
